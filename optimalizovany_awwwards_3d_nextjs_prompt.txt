The page should combine high-end typography, smooth scroll-driven motion, glassmorphism UI cards, premium micro-interactions, shader-based visuals, and carefully optimized performance.

The website must look and feel like an Awwwards-winning digital experience, but it must also remain buildable, maintainable, responsive, accessible, SEO-conscious, and production-ready.

5. Performance / Developer Experience

- Lazy loading for non-critical sections.

MANDATORY INTERACTION AND VISUAL EFFECT BLUEPRINT
You must strategically implement or plan the following 35 effects. Do not treat them as a random checklist. Explain where each belongs, why it exists, what technology controls it, and how to prevent performance or UX issues.

LAYER A: 3D SCENE & ENVIRONMENT
1. Liquid Glass Morphing
- Use refractive / transmissive glass-like materials on the primary 3D object.
- Use MeshPhysicalMaterial or custom shader material.
- Include clearcoat, transmission, thickness, roughness, envMap intensity, and subtle normal distortion.
- Mention fallbacks for low-end devices.

2. Procedural Light Grid
- Create an animated background plane or shader layer with grid lines, pulses, or scan-like movement.
- Use custom GLSL uniforms for time, intensity, scrollProgress, and sectionTheme.
- It should remain subtle and not compete with content.

3. Particle Attraction / Repulsion
- Create a particle system that reacts to mouse movement.
- Use BufferGeometry and shader uniforms where possible.
- The mouse should attract or repel particles inside the 3D scene.
- Include a performance cap and reduced-motion fallback.

4. 3D Tilt & Dynamic Glare
- The 3D canvas or selected objects should respond subtly to pointer movement.
- Use normalized mouse coordinates to influence rotation, camera parallax, and glare highlights.
- Avoid exaggerated tilt.

5. Floating / Levitation
- Add idle floating physics to hero objects using drei Float or custom sine-based animation in useFrame.
- Use tiny amplitude and slow movement for luxury feel.

6. Grain / Noise Overlay
- Add a subtle CSS/SVG film grain overlay over the viewport.
- Use pointer-events none.
- Keep opacity extremely low.
- It should add texture without reducing readability.

7. Ambient Color Shift
- Change scene lighting and background gradients according to active section.
- Use ScrollTrigger callbacks or central state to transition theme values.
- Include smooth interpolation rather than instant color jumps.

LAYER B: SCROLL-DRIVEN CINEMATIC MECHANICS
8. Assemble on Scroll
- 3D components or particles fly into place as the page scrolls.
- Use GSAP timelines connected to refs exposed from the 3D scene.
- Use scroll progress to control position, rotation, opacity, material intensity, and scale.

9. Scroll-Driven Warp
- Apply subtle vertex shader distortion, lens distortion, or camera FOV changes during fast scroll moments.
- It should create a premium kinetic feeling without nausea.
- Include reduced-motion fallback.

10. Smooth Page Transition
- Plan a route transition system with WebGL/GSAP overlay.
- Use a transition layer that fades, wipes, distorts, or blurs between routes.
- Keep it compatible with App Router limitations.

11. Scroll-Driven Blur & Slow Background Zoom
- Background layers should blur slightly and scale as depth changes.
- Use CSS transforms and filters for DOM backgrounds.
- Use GSAP for timeline control.

12. Infinite Unroll & Depth-Layered Parallax
- Text and DOM layers should move at different speeds relative to the 3D scene.
- Use transform3d, perspective, and ScrollTrigger scrub.
- Ensure content remains readable and does not feel chaotic.

LAYER C: MICRO-INTERACTIONS & HOVER STATES
13. Magnetic Hover
- CTA buttons should be gently pulled toward the cursor within a defined radius.
- Use pointermove and transform interpolation.
- Reset smoothly on pointerleave.

14. Mouse-Follow Glow & Glow Trace
- Implement a custom cursor or cursor glow layer that follows the pointer with easing.
- Optional trail effect via CSS radial gradient or canvas.
- Disable or simplify on touch devices.

15. Focus / Defocus on Hover
- When hovering over cards, the active card becomes sharper/brighter while sibling cards slightly blur, dim, or move backward.
- Use data attributes, group hover, Framer Motion, or CSS variables.

16. Elevation-Based Shadows & Glassmorphism
- Cards should respond to hover with shadow elevation, border glow, backdrop blur, and slight 3D tilt.
- Use tasteful frosted-glass design.
- Avoid excessive blur on low-end devices.

17. Staggered Reveal & Number Counter
- Use GSAP or Framer Motion to stagger section content reveals.
- Stats should count up from zero only when visible.
- Avoid counting repeatedly unless intentionally reset.

18. Shake / Wobble & Pixel Disintegration
- Add a refined glitch or disintegration hover state for special CTA buttons.
- Keep it premium, short, and rare.
- Use CSS masks, pseudo-elements, or canvas particles.
- Avoid making primary CTAs feel unstable.

19. Duotone / Gradient Map Image Hover
- Images or case-study thumbnails should shift to a duotone or gradient-map aesthetic on hover.
- Use CSS blend modes, filters, overlays, or SVG filters.

20. Smooth Accordion
- Implement ultra-smooth expandable panels using GSAP height animation.
- Measure natural height programmatically.
- Animate height, opacity, and internal content stagger.

LAYER D: HIGH-END TYPOGRAPHY & VECTOR ANIMATION
21. Revealing Text Mask
- Headlines and paragraphs reveal through clip-path or overflow-hidden masks.
- Use staggered line reveals.
- Respect reduced-motion settings.

22. Animated Gradient Text & Iridescent Text
- Use subtle gradient movement on selected words only.
- Avoid applying shimmer to all text.
- Ensure contrast remains accessible.

23. Text Scramble / Typewriter
- Use a character scrambling effect for section labels or short headings.
- Keep duration short and controlled.
- Avoid scrambling long paragraphs.

24. Marquee / Ticker
- Use an infinite horizontal ticker for brand values, services, or proof points.
- Pause or slow on hover.
- Ensure seamless looping.

25. SVG Stroke Draw & Morphing SVG
- Use SVG paths that draw themselves on scroll.
- Optional shape morphing between section milestones.
- Use GSAP DrawSVG if available or stroke-dasharray/stroke-dashoffset manually.

ARCHITECTURAL CONSTRAINTS
- Do not put heavy animation logic directly inside app/page.tsx.
- Do not use arbitrary global event listeners without cleanup.
- Avoid overusing useState for high-frequency animation data.
- Use refs, GSAP, useFrame, or external stores for animation-heavy data.
- Do not animate layout properties when transform/opacity can be used.
- Do not create excessive post-processing effects by default.
- Do not block first paint with heavy 3D assets.
- Do not rely only on WebGL for important text content.
- All important content must exist in semantic HTML.
- Respect touch devices and prefers-reduced-motion.
